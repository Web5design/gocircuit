The Go Circuit extends the Go Language and Runtime with the addition of a Spawn operator for spawning goroutines on remote hosts.

This small addition has far-reaching consequences:

  • Inter-process communication is type-safe and entirely done within the Go programming environment. No boiler plates. No RPC definitions. No DSLs. No special syntax. No non-Go code files.
  • Remote processes are linguistically identical to local goroutines. They can be manipulated in the same coding environment with the same linguistic tools Go programmers are already familiar with.
  • A single source file can implement a complex cloud application consisting of multiple clusters of differently purposed processes.
  • Familiar Go types and interfaces can be passed across runtimes by value or reference seamlessly. A global garbage-collector ensures proper cloud-wide semantics. Such features are instrumental to passing open files, streams, channels and sockets across processes in a correct and worry-free manner.
  • Sandbox-ed execution of a multi-application cloud system at the flip of a switch.
  • The circuit runtime acts as a light-weight distributed OS layer, which provides a uniform view of an entire cloud of applications in the form of a unified process file system with a myriad of introspection and administration facilities:
    • On-the-fly/in-production debugging and profiling
    • Issue tracking system
    • Administration scripting inside Go as well as on the Unix shell, via a carefully tailored set of command-line tools with familiar interfaces.
